Sans Docker - Installation & Utilisation

Pas d'overhead de virtualisation.

📦 Prérequis

☕ Java 17+

Télécharger: oracle.com/java
Vérifier: java -version

🏗️ Maven 3.8+

Télécharger: maven.apache.org
Vérifier: mvn -version

🗄️ PostgreSQL 15+

Télécharger: postgresql.org
Vérifier: psql --version

🚀 Démarrage Rapide

1

Vérifier PostgreSQL

Assurez-vous que PostgreSQL est en cours d'exécution sur votre système.

2

Lancer le script automatisé

Linux/macOS:

chmod +x start.sh && ./start.sh

Windows: Double-clic sur start.bat

3

Attendre le démarrage

Le script va vérifier les dépendances, créer la base de données et lancer l'application.

4

Accéder à l'application

Ouvrez votre navigateur à http://localhost:8080

👤 Identifiants de Connexion

Admin: login: admin | mot de passe: Admin@2024

Agent: login: agent1 | mot de passe: Agent@2024

📋 Démarrage Manuel (Alternatif)

Si les scripts automatisés ne fonctionnent pas:

Étape 1: Créer la base de données

psql -U postgres -f setup-db.sql

Étape 2: Compiler le projet

mvn clean package -DskipTests

Étape 3: Lancer l'application

mvn spring-boot:run -Dspring-boot.run.arguments="--spring.profiles.active=local"

⚖️ Avant vs Après

❌ Avant (avec Docker)

  • Installer Docker Desktop
  • `docker-compose up -d`
  • Conteneurs à gérer
  • Plus complexe à déboguer
  • Dépendance sur Docker
  • Temps de démarrage: ~30s

✅ Après (sans Docker)

  • Installer Java, Maven, PostgreSQL
  • `./start.sh` ou `start.bat`
  • Pas de conteneurs
  • Debugging natif avec IDE
  • Exécution native
  • Temps de démarrage: ~10s

🔧 Dépannage Rapide

PostgreSQL non trouvé

Voir SETUP.md pour installer PostgreSQL ou démarrer le service

Java non trouvé

Installez Java 17+ depuis oracle.com/java

Maven non trouvé

Installez Maven depuis maven.apache.org

Pour plus de dépannage, consultez SETUP.md (section Dépannage).
